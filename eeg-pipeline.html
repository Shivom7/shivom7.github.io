EEG analysis pipeline

A reproducible workflow for transforming raw electrophysiological recordings into quality-controlled features, visualizations and statistical analyses.

Documentation status: This page is a verified high-level scaffold based on the supplied CV. Installation commands, file names and example outputs will be added after the pipeline repository is supplied; no scientific implementation details have been invented.

Purpose

The pipeline supports multi-channel EEG research across meditation, attention and mind-wandering experiments. Its intended role is to standardize preprocessing decisions, preserve quality-control evidence and enable repeatable downstream analyses such as ERP and microstate modelling.

Research contexts

  • 8- and 32-channel EEG recordings
  • Audio-visual selective attention and switching
  • Mantra meditation and mind wandering
  • Age-related neural dynamics
  • Multimodal physiology, including ECG and GSR

Each stage should produce machine-readable outputs and a human-readable quality-control record. The final implementation diagram will map these stages to the actual functions and folders in the supplied repository.

Inputs & outputs

Expected inputs: raw EEG, event markers, channel metadata and participant/session metadata. Exact supported formats remain to be confirmed from the code.

Expected outputs: cleaned recordings, rejection and quality metrics, derived features, figures and analysis-ready tables. Exact schemas and naming conventions remain to be documented.

Reproducibility checklist

  • Environment and dependency versions pinned
  • Configuration separated from analysis logic
  • Raw data treated as immutable
  • Parameters and exclusions recorded per run
  • Deterministic random seeds where applicable
  • Quality-control plots retained with results
  • Participant identifiers handled without exposing personal data
